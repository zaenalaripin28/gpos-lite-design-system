Component Best Practices

✅ Do

  • Build components menggunakan design tokens
  • Dokumentasikan variants dan states
  • Test accessibility (keyboard navigation, screen readers)
  • Include focus states dan hover effects
  • Maintain consistent spacing dan sizing

❌ Don't

  • Jangan hardcode color values
  • Jangan buat custom component variations
  • Jangan skip keyboard interactions
  • Jangan mix component styles dari berbagai sources

Building New Components

Component Template

<!-- Follow this structure for new components -->

<!-- Base component -->
<div class="component">
  <!-- Content -->
</div>

<!-- Variants (primary, secondary, etc) -->
<div class="component component--primary"></div>
<div class="component component--secondary"></div>

<!-- States (hover, active, disabled, etc) -->
<div class="component:hover"></div>
<div class="component:active"></div>
<div class="component:disabled"></div>

<!-- Responsive variants -->
<div class="component md:component--alternative"></div>
