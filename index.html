Implementation Guide

Tiga cara mudah untuk menggunakan design system ini

01

Copy Tailwind Config

Gunakan tailwind.config.js yang telah dikonfigurasi dengan semua design tokens kami:

// tailwind.config.js
module.exports = {
  theme: {
    colors: { ... },
    spacing: { ... },
    shadows: { ... },
    // dan seterusnya
  }
}
02

Use Tailwind Classes

Semua design tokens sudah menjadi Tailwind utilities. Copy class names langsung:

<div class="bg-primary-500 text-white px-4 py-2 rounded-md shadow-md">
  Button
</div>
03

Reference CSS Variables

Atau gunakan CSS variables untuk custom styling:

.custom-element {
  background-color: var(--color-primary-500);
  padding: var(--spacing-4);
  border-radius: var(--border-radius-md);
  box-shadow: var(--shadow-md);
}

Best Practices

Panduan untuk menggunakan design system secara konsisten dan efektif

Do

  • Gunakan design tokens dari system kami, jangan hardcode nilai
  • Konsistenkan spacing menggunakan spacing scale (4px base unit)
  • Selalu gunakan semantic color naming (primary, success, error, etc)
  • Ikuti typography guidelines untuk readability dan hierarchy
  • Test responsive design pada semua breakpoints
  • Dokumentasikan component variant dan usage guidelines

Don't

  • Jangan hardcode warna, spacing, atau shadow values
  • Jangan buat custom color palette di luar design system
  • Jangan skip accessibility guidelines dalam component design
